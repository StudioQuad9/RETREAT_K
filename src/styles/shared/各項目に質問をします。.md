* スマホ表示の完成度

私の作るサイトは、基本的に、
スマホ表示をベースとします。
それが出来てから、PC版へSCSSを拡張します。
だから、このmixinが有効なのです。

```
// メディア・クエリ
//  sm => md => lg => xlの順になるようスタイリングをレイアウトすることが秘訣です。
// 『min-width』の設定を『map（js的で良い）』に保存する。
$breakpoints: (
  // 480px以上 => スマホを横向きにした時の設定
  "sm": "(min-width: 480px)",
  // 600px以上 => タブレットを縦向きにした時の設定
  "md": "(min-width: 600px)",
  // 960px以上 => ノートPCの設定
  "lg": "(min-width: 960px)",
  // 1280px以上 => デスクトップPCの設定
  "xl": "(min-width: 1280px)",
);

@mixin mq($bp) {
  @media #{map.get($breakpoints, $bp)} {
    @content;
  }
}
```

ですから、完成度としては、トップ・インデックス・詳細とそれぞれほぼ100％を目指します。
ただし、材料として写真がないことは前提です。
それはアタリで解決します。
ちなみにあなたのいう完成度とは、何を言いたいのですか？
私に何を言わせようとしてますか？
